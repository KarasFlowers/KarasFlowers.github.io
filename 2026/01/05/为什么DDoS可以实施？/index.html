为什么DDoS可以实施？
Keep Team Lv3
  2026-01-05 12:00:00   2026-05-09 14:27:55  
  • 编程迷思

暴露自己的IP地址不是个好选择，其中之一就是可能受到DDoS（Distributed Denial-of-Service）打击。简单类比，相当于大量人员去饭店只占座不点餐。这个“座”是什么？DDoS究竟占了服务器的什么资源？为什么听上去如此简单的攻击到现在还是问题（吗）？

DDoS原理

在计算机网络中（特别是 TCP 协议），服务器和用户建立连接是有成本的。
TCP的三次握手：

  1. 客户端向服务器发送SYN包（我要吃饭）
  2. 服务器恢复SYN-ACK包（好的，这是你的桌号，请确认）
  3. 客户端回应ACK包（确认入座）

DDoS执行了很多第一步，服务器为每一个请求分配内存，记录下“有个家伙要来，IP是xxx”，然后回复 SYN-ACK（第二步）。可是DDoS并不回应ACK包（或者黑客伪造了假的 IP 地址，服务器的回信寄给了空气）（第三步）。

服务器为了等待那个永远不会回来的 ACK，必须在内存中维护一个“半连接队列” (Semi-open connection queue)。每记录一条“等待确认”的连接，就需要消耗几十到几百字节的内存，一旦到达最大容量，正常的请求就会被忽视。

除了“占座”（耗尽连接资源），DDoS 还有另外两种常见的“捣乱”方式，也可以用饭店类比：

1. 堵门（带宽消耗型 - Volumetric Attacks）

  • 类比：这帮流氓不进店占座，而是叫了 1000 个人挤在饭店大门口及走廊里。

  • 原理：利用海量的垃圾数据包（UDP Flood, NTP Amplification 等）塞满你服务器的网络带宽。

  • 后果：真正的客人连饭店大门都挤不进去。

  • 资源：占用的不是服务器内部的资源，而是**网络管道的容量 (Bandwidth)**。

2. 累死厨师（应用层攻击 - Application Layer Attacks）

  • 类比：流氓进店了，也确实点餐了。但他一个人点了“满汉全席”，还要厨师每道菜都现做，做完了一口不吃，倒掉重做。

  • 原理：攻击者针对消耗 CPU 或数据库最严重的页面发起请求（比如 CC 攻击）。例如疯狂调用“搜索全站历史记录”这种极耗算力的功能。

  • 后果：前台（网关）可能觉得没满座，但后厨（CPU/数据库）已经累吐血了，无法服务其他人。

  • 资源：占用的是 CPU 计算能力 和 磁盘 I/O。

应对方法

之所以叫分布式，是因为黑客控制了全球各地几万台中了病毒的电脑（肉鸡）、智能冰箱、摄像头等一起攻击。

  1. 对于“占座不点餐”，可以采用动态调节等待时间、异常间隔请求阶段等方式延缓。
  2. 对于“堵门”（发送大量垃圾数据包）：“分流(CDN)”与“扩建”，将压力分散到全国各个节点；提高总容量。
  3. 对于“累死厨师”：WAF (Web 应用防火墙)：这是由 AI 把守的大门。它会分析行为模式：“这个 IP 一秒钟点了 50 次‘红烧狮子头’，人类不可能手速这么快”，于是直接封锁 IP；人机验证 (CAPTCHA):最经典的证明自己是人类；限流：顾名思义

为什么泛滥

成本极低，暗网有专门的服务。且可以称为勒索手段、恶意竞争等。
