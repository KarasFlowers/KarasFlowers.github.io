为什么能够切换不同的ai模型编程？
Keep Team Lv4
  2026-01-05 12:00:00   2026-05-09 17:01:18  
  • 编程迷思

在 Cursor、Copilot 这些 AI 编程工具里，你可以随时切换不同的 AI 模型——今天用 GPT-4，明天用 Claude，后天用 Gemini。为什么能这样？它们不是不同的公司做的吗？

统一的接口：OpenAI API 格式

答案是：它们都遵循同一套 API 格式。

OpenAI 最早定义了 Chat Completion API 的格式：

这个格式太成功了，以至于几乎所有 AI 厂商都兼容了这个格式：

  • OpenAI：GPT-4、GPT-3.5
  • Anthropic：Claude
  • Google：Gemini
  • 开源模型：LLaMA、Mistral（通过 vLLM、Ollama 等部署后也兼容这个格式）

类比：不同品牌的手机都用 USB-C 接口充电。不是因为 USB-C 最好，而是因为大家都约定了这个标准。

本地模型也能接入

通过 Ollama、vLLM 等工具，你可以在本地部署开源模型，它们也暴露兼容 OpenAI 格式的 API：

所以 Cursor 不仅能接云端模型，也能接本地模型，只要你给它一个兼容的 API 地址。

这意味着什么？

  1. 不被锁定：你不需要绑定某一个 AI 厂商
  2. 自由切换：哪个模型好用就用哪个
  3. 成本可控：可以用便宜的模型做简单任务，贵的模型做复杂任务
  4. 隐私保护：敏感代码可以用本地模型处理

这就是标准化的力量——当所有人都遵循同一个接口标准时，用户就有了选择权。
